⏺ I have enough info. Here's the complete list of files you need to build a
  Philadelphia cityscape model, based on tests/chicago/run.sh and the modelgen
  CLI in ModelGenerator.cpp:225-261.

  Required inputs for a new city (Philadelphia)

  1. City boundary shapefile (--shape, --dbf)

  A polygon shapefile defining the city limits. You need all four sidecar files
  (the tool only takes .shp + .dbf as args, but .shp needs the others to open):
  - philadelphia.shp — geometry
  - philadelphia.dbf — attributes
  - philadelphia.shx — index
  - philadelphia.prj — projection (must be WGS 84 / EPSG:4326)

  Source: city open-data portal (OpenDataPhilly has a "City Limits" layer).
  Reproject to WGS84 with ogr2ogr -t_srs WGS84 out.shp in.shp if it isn't
  already.

  2. OSM XML (--osm-xml)

  - Philadelphia.osm — buildings + streets

  Source: https://download.bbbike.org/osm/bbbike/ (pick or request a
  Philadelphia extract; download the .osm.gz and gunzip it).

  3. PUMS microdata (--pums-h, --pums-p) — Pennsylvania, since Philadelphia is 
  in PA

  - psam_h42.csv — household records (state code 42 = PA)
  - psam_p42.csv — person records

  Source: https://www.census.gov/programs-surveys/acs/microdata/access.html.
  Pick a vintage (e.g., ACS 2022 5-year).

  4. PUMA shapefile (--puma-shp, --puma-dbf) — same vintage as PUMS

  - puma_pa.shp
  - puma_pa.dbf
  - puma_pa.shx, puma_pa.prj (sidecars)

  Source: https://www.census.gov/cgi-bin/geo/shapefiles/index.php (state-level
  PUMA layer for Pennsylvania). Reproject to WGS84:
  ogr2ogr -t_srs WGS84 puma_pa_WGS84.shp puma_pa.shp
  Per the README, use the same year for PUMS and PUMA.

  5. Model adjustments file (--adjust-model, optional but recommended)

  - philadelphia_adjustments.txt — you author this. Copy
  tests/chicago/chicago_adjustments.txt and tune HomeBuildingTypes,
  ApartmentBuildingTypes, IgnoreBuildingTypes, DefaultApartmentLevels for
  Philly's OSM tagging quirks. Also lists OSM building IDs to ignore.

  6. Optional: LandScan population GIS (--pop-gis)

  - lspop20XX/w001001.adf (plus the rest of the ESRI grid sidecars: .adf,
  dblbnd.adf, hdr.adf, etc.)

  LandScan was dropped in the Jan 2025 Chicago run (see run.sh:24) — runs are
  slower without it but it's no longer required.

  ---
  Minimum invocation

  ../../modelgen \
    --shape boundaries/philadelphia.shp \
    --dbf   boundaries/philadelphia.dbf \
    --osm-xml Philadelphia.osm \
    --adjust-model philadelphia_adjustments.txt \
    --pums-h pums/pa_2022_5yr/psam_h42.csv \
    --pums-p pums/pa_2022_5yr/psam_p42.csv \
    --puma-shp pums/puma_pa_2020_WGS84.shp \
    --puma-dbf pums/puma_pa_2020_WGS84.dbf \
    --pums-h-cols SERIALNO \
    --xfig philadelphia.fig --scale 16384000 \
    --out-model philadelphia_model.txt > bld_info.txt

  Put everything under tests/philadelphia/ mirroring the tests/chicago/ layout.

⏺ You need two zips:

  - csv_hpa.zip (38M) — Pennsylvania household records → extracts to
  psam_h42.csv (the --pums-h input)
  - csv_ppa.zip (87M) — Pennsylvania person records → extracts to psam_p42.csv
  (the --pums-p input)

  Naming convention on this FTP

  - csv_ vs sas_ → file format. Use csv_ — the modelgen reads CSV
  (--pums-h/--pums-p are CSV paths). Ignore the sas_ files.
  - Next letter: h = housing, p = person. You need both.
  - Last two letters: state postal code. pa = Pennsylvania.
  - us = entire country (don't grab these, they're huge: 908M / 2.1G).

  Don't confuse

  - csv_phi.zip (10M) is not Philadelphia — there's no per-city PUMS. The phi
  here is Person + Hawaii ("hi"). Philadelphia is covered inside csv_hpa.zip /
  csv_ppa.zip, and you'll narrow to Philly later via the PUMA shapefile (PUMA
  IDs intersecting the city boundary).

  After download

  unzip csv_hpa.zip   # yields psam_h42.csv (+ docs)
  unzip csv_ppa.zip   # yields psam_p42.csv (+ docs)
  State FIPS 42 = PA, which is why the files inside are named with 42.

  Make sure the PUMA shapefile you grab from the TIGER shapefile site is the
  same vintage (1-yr vs 5-yr / same release year) as whatever PUMS release this
  FTP directory corresponds to — mismatched years will have PUMA IDs that don't
  line up.
